## 一般模块

### 主模块

> 用于导出的模块，即模块入口

+ forward为`x`,`y`

### 训练模块DS_BLOCK

> 真正用于训练的单位

+ 无需改名

+ forward变量名为x,y

> 以下是用于修改的模块

###  1. 特征处理模块(DGCNN_MAX_Block)

+ forward变量为`feature`
+ 常客是`get_graph_feature`方法，是让我们将一对的坐标系进行整理的好朋友
  
-[ ] `get_graph_feature`方法学习

### 2. 常规的处理模块（OANet）

> 这类模块的顺序在`游玩时`我们随意摆放

+ forwar变量为`data`或`x`

### 3. 全局处理模块(GCN_Block)

> 最难进行处理的模块！！！，因为他涉及到很广的区域，如果说我们可以将这块进行改写了，说明发刊就不远了

+ forward参数`x`(`data`)和`w` 

### 我们自定义的模块

> 自己写的用于特殊用途的模块，这类模块我们需要详细的书写运行原理和用途

+ forward的参数自己定


## 运动向量类

### 向量

+ forward变量名为x,motion

## 卷积

### 卷积序列`torch.Sequential`

> 上一个模块的输出是下一个模块的输入，目前来看是要单输入，单输出

+ 常用的方式是将数据进行高维训练，减少训练时的平滑或者急剧变化，激活，残差和二维卷积是常客

## 建议

1. 买一个性能好点的电脑，现存达到6g
2. 一般我建议新手修改模型的时候从卷积序列入手，因为他输入输出的要求比较简单，然后他的结果是一个`正方形`
    > 这里说明下`正方形`:
    > 
    > 就是说输入是nxn，输出也是nxn这钟款式
    >
    >  经典的方法：
    >
    >  nn.Conv2d(self.in_channel, self.in_channel, (1, 3))
   
## 常用的套路

1. 将卷积过后的2*n通道张量平分成两份n通道张量分别进行处理
2. 如果我们遇到模型初始化的时候有很多参数的问题我们看看原来的模型有给的变量，如果说是一个数字，那么我们就改数字，如果是张量我们就根据现有的张量（out,w）编一个
